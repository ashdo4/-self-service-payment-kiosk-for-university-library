Accepted Notes of the kiosk are Rs. 1000, Rs. 500, Rs. 200 and Rs. 100. Only one note can be inserted per 
transaction. Change Available from the kiosk is Rs. 500, Rs. 200, Rs. 100, Rs. 50 and Rs. 20
The Rules are 
The user selects one service. 
If the service is a fine payment, the user enters the number of overdue days. If the service is a book 
reservation, the user enters the number of books reserved (a maximum of 3). Note that the name of the 
book or other details of the book are not necessary. 
The system calculates the total fee. 
The user should insert the money 
If insufficient money is inserted: 
 Transaction is cancelled. 
If the requested service is unavailable: 
 Transaction is cancelled. 
Change returns using available denominations. 
A denomination is used only once when returning change. 
If exact change cannot be produced: 
 The inserted note is returned. 
 "Unable to Return Change" message displayed. 
After a successful transaction: 
 Service inventory decreases. 
 Change inventory decreases. 
After every transaction, display: 
 Remaining reservation slots 
 Remaining card replacements 
 Remaining renewals 
 Remaining quantities of all notes/coins 
The updated service inventory and change inventory is considered for the next transaction. 
The program run on a set of data until the user quits
